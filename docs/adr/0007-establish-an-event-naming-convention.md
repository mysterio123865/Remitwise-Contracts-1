# ADR 0007: Establish an event naming convention

- Status: Accepted
- Deciders: arisu6804

## Context

The RemitFlow smart contract needs a clear, documented approach to "establish an event naming convention" so the codebase stays consistent and auditable.

## Decision

We establish an event naming convention as the standard for this contract, in line with Soroban best practices.

## Consequences

Improves clarity, testability, and maintainability, and gives future contributors a recorded rationale to build on.
