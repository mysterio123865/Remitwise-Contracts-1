# ADR 0011: Introduce an admin role

- Status: Accepted
- Deciders: arisu6804

## Context

The RemitFlow smart contract needs a clear, documented approach to "introduce an admin role" so the codebase stays consistent and auditable.

## Decision

We introduce an admin role as the standard for this contract, in line with Soroban best practices.

## Consequences

Improves clarity, testability, and maintainability, and gives future contributors a recorded rationale to build on.
